# Session 2026-05-05 · 修复 rule 提交报"placeholder passage 不存在"（第 04 次）

## 1. 用户输入

> 「Error: the passage "placeholder" does not exist，在我点击了 rule 的提交后，报错了」

## 2. 根因分析

ruleWorksheet 里：

```twee
<<textbox "$ruleInputs[_key]" "" placeholder "缺失字段">>
```

SugarCube 的 `<<textbox>>` 语法是：

```
<<textbox receiverName defaultValue [passage] [autofocus]>>
```

它**只**有 4 个位置参数，不认 `placeholder` 关键字。所以解析结果：

| 位置 | 实际值 |
|---|---|
| receiverName | `"$ruleInputs[_key]"` |
| defaultValue | `""` |
| passage（导航目标） | `"placeholder"` |
| autofocus | `"缺失字段"` |

当用户在 textbox 里按 Enter（SugarCube `<<textbox>>` 的默认 commit 行为）→ 它去 `Engine.play("placeholder")` → 找不到 → 抛"the passage placeholder does not exist"。

类似的 `<<button "提交校验" class "rule-submit">>` 同样是错语法（`<<button>>` 也不认 `class`），如果用户先点提交也会报"passage class does not exist"。

## 3. 修复

放弃 SugarCube 自己的 `<<textbox>>` / `<<button>>`，全部改为原生 HTML，避免被 wikifier 当成命名参数误解析；交互绑在 `:passagerender` 的 click/input handler 上。

### `Widgets.twee` 的 `<<ruleWorksheet>>`
```twee
<input type="text" class="rule-input" placeholder="缺失字段"
       @data-rule-key="_key"
       @value="setup.ruleInputValue(_key)">
...
<button type="button" class="rule-submit" data-rule-submit="1">提交校验</button>
```
- 用 `@attr="expr"` 让 SugarCube 求值；普通字面量直接写在 HTML attribute
- 提交按钮换成原生 `<button>`，避免 `<<button>>` 的 passage 误参

### `StoryInit.twee`
- 新增 `setup.ruleInputValue(key)`：从 `$ruleInputs` 读值并兜底成空字符串
- 新增 `setup.submitRules()`：遍历 `setup.rules.answer`，比对 trim 后的输入；命中显示"规则碎片已被接受"，否则"部分字段与已恢复档案冲突"。最后 setScreen + persist + pushBrowser
- `:passagerender` 增加两个 handler：
  - `[data-rule-key]` 监听 `input` → 同步到 `State.variables.ruleInputs[key]`
  - `[data-rule-submit]` 监听 `click` → 调 `setup.submitRules()` → `Engine.play("Game")`

### 测试
- 新增 smoke 用例：进入 rule 表 → 直接点提交（不填）→ 期望 body 包含"部分字段与已恢复档案冲突"，**不允许 page error**（错误会被 page.on("pageerror") 捕获，最终 errors 检查处会失败）

## 5. 教训

- **SugarCube 宏不认 named/keyword 参数**。`<<textbox>>` `<<button>>` 等只接受位置参数。看到 `placeholder "..."`、`class "..."` 这种语法当场就要警觉——它们会被解析成第 N 位的 passage 名/路径，bug 在 hot-path 才爆
- **碰到要传自定义 attr / class / placeholder 的场景，直接用原生 HTML + `@attr="expr"` directive + jQuery handler**，比挣扎着把 SugarCube 宏拼起来稳得多
- **smoke test 必须覆盖每一个交互 surface**。这次 bug 在线下被用户撞到，是因为之前的 smoke test 完全没点过 rule 提交按钮。新增的"开 rule 表 + 提交"用例从此把这条路径锁住

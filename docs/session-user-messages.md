### 游戏类型
做一个类似 Type Help 的纯文字解密游戏，玩家通过故事线索发现、挖掘、探索故事，最终解开谜题。核心玩法：通过输入框输入指令获取线索信息，不仅故事需要玩家推理解读，指令本身也需要玩家通过故事获取。

### 游戏一：校园规则怪谈
背景故事：

以"规则怪谈"为核心，融合"克苏鲁神话"作为背景。讲述发生第一学院的故事，你收到开学典礼邀请函，随着游戏进展获取一个存放在机房中脱机保存的备份硬盘（已损毁），修复后通过关键词搜索查找部分残存资料，通过档案资料查阅一层层揭开学校曾发生的故事与谜题。

**订正描述**：许久之前，在第一学院中流行起了校园怪谈的故事。因为当时师生压力大环境压抑，这些故事便成为了大家茶余饭后的消遣。一开始只是口口相传的故事与流言蜚语，后来的故事也越发荒诞——被霸凌者死后的怨念会在午夜 2 点找上进入体育馆的人，和长着龅牙的女生聊天会在当天晚上被咬死。起初大家并不在乎真相如何，只是觉得好玩。可随着流言越来越多，那些曾经只是停留在师生耳边的故事却似乎成为了真的规则。恐惧与哀嚎、痛苦与悲鸣笼罩这所学校，这些情绪又逐渐招致了更多邪恶存在，死者与规则也越来越多。而我们就是邪恶存在。

## 13. 进入页面（主菜单）详细描述（09:55）
之后我们来写进入页面，刚刚那个是过场。这里是进来后默认的位置，有**三个按钮**：

### 关于游戏
点击"关于游戏"跳转到"初始化"中（也就是刚刚的过场页面）。

### 故事
"故事"表示一个作品，每个故事之间相互独立。我们的"第一学院"其实只是一个故事。

### 档案
"档案"就是**存档**。

**点击档案后**：在左侧跳出侧边栏，显示游玩过的故事。点击故事，弹出二级侧边栏，显示保存的存档。游戏会自动存档——每次执行完成一次写入命令或查阅文档时自动存档，默认逻辑是覆盖。感觉就像是真的档案库一样。逻辑可以参考 Type Help 的存档逻辑，只是换了个 UI。

**点击新游戏后**：类似逻辑，在侧边栏展开可选故事。点击故事弹出侧边，这里也是**打字机效果**弹出关于这个故事的描述。点击可以直接跳过打字机效果。选中故事后有两个选项：「开始新游戏」和「继续游戏」。

注意：对玩家的存档和自动存档、自定义存档做**体验优化**，不要让玩家的存档覆盖或损坏。

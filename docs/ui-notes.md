# UI 骨架说明

当前 UI 参考草稿图，仅实现结构，不锁定最终画风。

## 视图

- 首页：显示标题、简介、开始新故事、档案终端、关于游戏。
- 主控制页：占满浏览器的内容面板，里面显示当前命令结果和底部居中输入框。
- 终端：黑底单屏结果，用于 `help`、`find`、错误信息和系统反馈。
- 档案：显示编号、类型、章节、标题、正文和关联档案。
- 规则表：填写规则怪谈缺失字段。
- 笔记和关联：用于 `note`、`link` 命令。

## 交互规则

- 输入框按 Enter 提交，不使用提交按钮。
- 每次命令刷新为新的当前页面，不拼接检索历史。
- 当前命令显示在终端第一行，例如 `> help`。
- 首次进入显示纯介绍页；介绍页没有输入框，文字以打字机效果逐行显示。
- 介绍页点击一次会跳过当前页打字机效果；当前页全部显示后再点击进入下一介绍页。
- 第二页及之后显示左上返回按钮，可回到上一介绍页。
- 介绍流程完成后进入主系统；之后重新打开浏览器会自动继续到上次页面。
- `back` 和浏览器返回按钮会尽量回到本次会话的上一条命令页。
- 存档使用项目自己的 `localStorage` 自动保存，不使用 SugarCube 默认 `AUTOLOAD` 弹窗。
- 开发阶段可输入 `reset` 清除进度并重新播放首次介绍。

## 后续可扩展

- 左侧故事列表可按章节折叠。
- 档案列表可加入“损坏/已恢复”状态。
- `link` 可升级为可视化关系图。
- 最后一条规则可加入“无论填什么都会成为真实规则”的特殊演出。
